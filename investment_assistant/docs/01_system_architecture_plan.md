# 個人智能理財助理 (Investment AI Assistant) 規劃書

## 1. 系統定位與目標
這是一個自動化的「專屬理財幕僚」，主要針對您在筆記中提到的 ETF、個股（台/美）以及總經局勢，每天早晨為您準備好當日的投資決策參考。
*   **核心功能**：每日自動抓取台/美股數據與總經指標，結合最新新聞，透過大語言模型 (LLM) 進行分析總結，並推送推薦與警告給您。
*   **追蹤重點**：
    *   **投資標的**：國防科技 (SHLD)、半導體 (SMH, 892)、台股 5G (881)，以及 AI 基礎建設 (能源/電網)、數位孿生、智慧製造等未來趨勢。
    *   **基本面指標**：個股的 EPS、本益比、自由現金流、淨利率、負債比。ETF 的內扣費用、追蹤誤差、折溢價、殖利率與流動性。
    *   **總體經濟**：實時失業率、聯準會 (Fed) 利率動向與貨幣政策態度。

## 2. UI 介面與推播平台建議 (LINE vs Telegram vs App)
您提到「建議做成 App 嗎？UI 透過啥好？」
**結論：強烈建議先做成「通訊軟體聊天機器人 (Bot)」，目前不建議直接開發 App。**

對於「每日資料推送與提醒」的需求，通訊軟體的觸及率和開發效益是最高的。

| 平台 | 優點 | 缺點 | 綜合評價 |
| :--- | :--- | :--- | :--- |
| **Telegram Bot** | 1. **完全免費無限制** API。<br>2. 開發極度容易。<br>3. 支援互動按鈕，介面簡潔快速。 | 在台灣較少人用作日常通訊（但對個人理財專用完全無影響）。 | ⭐⭐⭐⭐⭐ (對於個人開發者最首選) |
| **LINE Bot** | 1. 開啟率最高，符合台灣人習慣。<br>2. 可製作圖文選單 (Rich Menu) 讓操作介面很像 App。 | 免費版推播訊息有數量限制 (每月約 200 則上限)，超過需付費。 | ⭐⭐⭐⭐ (適合想做精美選單者) |
| **Discord Bot** | 可建立不同頻道分類 (如：新聞區、報價區、總經區)，Webhook 簡單。 | 手機版閱讀體驗較雜亂，偏向社群討論而非個人專屬看板。 | ⭐⭐⭐ |
| **自建 App/Web** | 自由度最高，可做華麗的數據儀表板 (Dashboard) 和互動圖表。 | 開發成本極高包含前後端、上架，且需自行處理推播通知 (Push Notification)。 | ⭐⭐ (建議未來進度成熟後再考慮) |

> [!TIP]
> 建議先從 **Telegram Bot** 開始起步。之後如果想將資訊結構化成圖表，可以結合簡易的 Web 網頁（例如使用 Streamlit 或是 Next.js 建立一個專屬於您的 Dashboard），然後由 Bot 將網頁連結推給您。

## 3. 系統架構設計建議（低成本 / 無伺服器方案）
您提到目前擁有 Google 的 LLM 資源。我們可以用極低成本的方式把各個積木拼湊起來：

*   **資料收集層 (Data Crawling)**
    *   **台股與基本面**：使用 Python (`requests` / `BeautifulSoup`) 定時爬取 Goodinfo，或使用現成的開源套件 (如 `FinMind`) 獲取資料。
    *   **美股/ETF/總經**：使用免費的 `yfinance` 獲取美股/ETF報價與歷史數據，使用 FRED API 獲取美國失業率與聯準會基準利率。
    *   **新聞資訊**：利用 RSS Feed 或 Google News API 爬取關注的產業新聞 (如能源網、台積電等)。
*   **智能分析大腦 (AI Engine)**
    *   **模型選擇**：您可以申請 **Google Gemini API** (目前開發者有免費額度)。程式抓完資料後，透過 API 發送給 Gemini。
    *   **Prompt 設計 (提示詞)**：將您筆記中的規則教給 Gemini。例如：「如果 SMH 折溢價大於 1%，請在報告中亮紅燈提示等待回落」、「分析今日收集的失業率數據判斷是偏鷹還偏鴿」。
*   **自動化排程與部署 (Automation)**
    *   **GitHub Actions**：**強烈推薦！** 完全免費的雲端排程工具，可以設定每天早上 8:00 自動執行 Python 爬蟲與傳送訊息，不用開著電腦。
    *   **雲端主機 (VPS)**：如果要讓 Bot 能隨時回覆您的問題 (雙向互動)，可以租用便宜或免費的雲端主機 (如 AWS EC2 Free Tier 或 GCP e2-micro)。

## 4. 開發實作藍圖 (Phases)

建議我們分階段來完成，不要一次寫太複雜的需求：

*   **Phase 1: 單向推播 MVP (最小可行性產品)**
    *   建立一個 Telegram Bot (取得 Token)。
    *   撰寫 Python 腳本：用 `yfinance` 和簡易爬蟲抓幾個重點 ETF (如 SMH, 881) 的價格，並抓取一兩則重點新聞。
    *   透過指令呼叫 Google Gemini API 產生 100 字短評。
    *   設定 GitHub Actions 每日早上定時發送到 Telegram。
*   **Phase 2: 深度數據整合**
    *   加入 Goodinfo 的 EPS、本益比、殖利率爬取邏輯。
    *   加入美國失業率與 Fed 利率數據抓取。
    *   優化 Gemini 的 Prompt，讓它產出的報告具有您筆記中的「防護網思維」(例如檢視 FCF、D/E 比率)。
*   **Phase 3: 雙向互動式助理**
    *   將系統部署到常駐伺服器或 Vercel/Cloud Functions。
    *   加入對話功能，例如您可以在聊天室輸入 `@bot 幫我查 2330 淨利率`，Bot 現場爬資料交給 LLM 分析後回報給您。
